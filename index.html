API-Einstellungen

Verbinde Egomorph Core mit einem externen LLM. Funktioniert mit OpenAI, OpenRouter, Ollama (http://localhost:11434), LM Studio, llama.cpp und jedem OpenAI-kompatiblen Endpunkt.

Hinweis: API-Keys werden im Browser gespeichert. Für öffentliche Deployments ist ein eigener Proxy sicherer als ein direkt eingetragener Key.

Gilt nur für API-Antworten. Höhere Werte verhindern abgeschnittene Antworten.

Beispiel-Konfigurationen:

Codex-Gateway

Nutzt das lokale Egomorph-Core-Gateway zur offiziellen Codex-CLI mit persistentem Codex App Server und Streaming. Melde dich einmal mit ./egomorph codex login oder ./egomorph codex login --device-auth an und starte danach ./egomorph dashboard oder ./egomorph gateway.

Die Bridge ist nur fuer lokale, vertrauenswuerdige Nutzung gedacht. Keine ChatGPT-Cookies, Access-Tokens oder auth.json in die App kopieren.

Einrichtung in 3 Schritten
  1. Einmal anmelden: ./egomorph codex login oder auf Termux/headless ./egomorph codex login --device-auth.
  2. Gateway starten: ./egomorph dashboard oder ohne Browserstart ./egomorph gateway.
  3. Hier speichern und mit Status prüfen kontrollieren.

Nach erfolgreichem Login bleibt Codex lokal angemeldet. Das Gateway haelt laufende App-Server-Sessions; Egomorph Core speichert nur Bridge-URL, Modell und Tokenlimit.

Gilt nur fuer den Codex-Modus. Der Wert wird als Laengenhinweis an den App-Server uebergeben.
